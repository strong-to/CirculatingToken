## ADDED Requirements

### Requirement: 持久化 Launchpad 各步骤数据
在用户停留于 Launchpad 页面期间，多步骤表单 SHALL 保留第一至第六步的所有输入（文本、选择、上传），无论通过按钮前进/后退还是点击步骤列表跳转。

#### Scenario: 返回前序步骤不丢失数据
- **GIVEN** 用户已经在多个步骤中输入内容、上传文件并完成选择
- **WHEN** 用户前进后又通过步骤条或按钮返回任意已完成步骤
- **THEN** 该步骤内所有字段会自动回填之前保存的值，无需重新输入。

#### Scenario: 页面重新挂载仍能恢复
- **GIVEN** 用户已经在流程中填写了数据
- **WHEN** 页面因软刷新或瞬时路由切换而重新挂载
- **THEN** 内存状态会立即恢复最新记录，页面不会出现空白输入框。

### Requirement: 第三步上传反馈
第三步上传区在文件成功上传后 SHALL 呈现明确的成功状态：显示文件名、成功提示以及默认描述文案；若解析失败也要保持可见内容。

#### Scenario: 上传成功显示确认信息
- **GIVEN** 用户上传受支持的文件
- **WHEN** 上传成功完成
- **THEN** UI 展示文件名、成功提示与默认说明文字，并在用户离开再返回第三步时维持该状态。

#### Scenario: 上传失败给出可行动提示
- **GIVEN** 文件上传失败
- **WHEN** 错误发生
- **THEN** UI 弹出清晰的错误文案并提供再次上传的入口，无需刷新页面。

### Requirement: 第七步只读预览
第七步 SHALL 以只读方式汇总展示第一至第六步的最新数据（文本、图片、文件、数值），不再渲染可编辑表单。

#### Scenario: 预览实时映射已填数据
- **GIVEN** 用户已完成或部分完成前六步
- **WHEN** 进入第七步
- **THEN** 预览应同步展示对应文本、LOGO/文档、令牌与经济配置等信息，并在上游数据更新时即时反映。

#### Scenario: 预览对缺失字段友好提示
- **GIVEN** 某些步骤尚未填写完整
- **WHEN** 进入第七步
- **THEN** 预览用“未提供”等占位文案替代空白输入，便于用户识别待补内容。
